# 01 · 需求规格

| 项 | 值 |
|----|----|
| 文档版本 | v0.1 |
| 状态 | 已按旧 HTML 原型重构 |
| 事实源 | `image-gen-refactor.html`、用户要求、`Onezzr/ssr` |
| 项目类型 | 纯前端 AI 图片生成工具 |

## 1. 背景与目标

旧实现是一个 8000 多行的单文件 HTML，包含完整界面和大量功能，但后续 AI 难以接手，且存在未定义函数、错误函数名、按钮结构被覆盖、缺失图片资源等问题。

本次目标是在尽量保持原界面和功能的前提下，重构成可维护的静态前端项目，并补齐完整文档。

## 2. 产品定位

一个浏览器直连 OpenAI-compatible 图片生成 API 的本地工具，适合个人配置 API Key、输入提示词、批量生成图片、保存图片展馆和导出数据。

## 3. MVP 范围

### 本期做

- 保留“画图 / 图片展馆”双 Tab。
- 保留 API 配置管理、模型获取、当前配置展示。
- 保留文生图、图生图、批量生成、取消、进度统计。
- 保留参考图片压缩和缩略图管理。
- 保留提示词历史、快速示例。
- 保留 IndexedDB 图片展馆、筛选、排序、分组、预览。
- 保留数据导入/导出、批量下载、清空数据、自动下载。
- 保留网络快速检测和完整诊断。
- 修复旧原型中已经确认的运行时 bug。
- 写清文档，让后续 AI 能按事实源接手。

### 本期不做

- 不接入自有后端。
- 不处理真实 API 提供商的计费、限流、账号管理。
- 不把 API Key 写入项目文件。
- 不改成 React/Vite，保持零依赖静态前端。

## 4. 功能需求

| 编号 | 功能 | 说明 |
|------|------|------|
| FR1 | API 配置管理 | 新增、编辑、删除、启用配置；同步到生成表单 |
| FR2 | 模型列表获取 | 调用 `GET /v1/models`，写入配置表单的模型候选 |
| FR3 | 图片生成 | 调用 `POST /v1/responses`，从流式或 JSON 响应中提取图片 |
| FR4 | 批量生成 | 支持多数量逐张生成、失败计数、成功计数、取消 |
| FR5 | 图生图 | 上传参考图后压缩并作为 `input_image` 发送 |
| FR6 | 提示词管理 | 快速示例、历史保存、复用、置顶、删除 |
| FR7 | 图片展馆 | 保存生成记录并支持预览、删除、筛选、排序、分组 |
| FR8 | 数据管理 | 导出/导入全部数据、批量下载图片、清除所有图片、清空数据 |
| FR9 | 网络诊断 | 检测在线状态、Base URL 格式、模型接口延迟和错误 |
| FR10 | 接手文档 | README、docs、CLAUDE 说明结构、边界、验证 |

## 5. 非功能需求

| 编号 | 要求 |
|------|------|
| NFR1 | 纯静态运行，不需要构建步骤 |
| NFR2 | 主逻辑拆分到 `assets/js/app.js`，避免继续堆进 HTML |
| NFR3 | 关键操作失败必须显示用户可理解的状态 |
| NFR4 | API Key 不进入仓库，只保存在用户浏览器本地 |
| NFR5 | 移动端可用，至少保持单列布局和可点击控件 |
| NFR6 | 后续 AI 能通过 docs 理解功能树、文件职责、数据流 |
| NFR7 | 生成过程中保持页面可滚动、可取消；非当前页的大图网格不得反复重建 |

## 6. 验收标准

- 打开 `index.html` 能看到画图页和图片展馆页。
- 添加 API 配置后，当前配置显示和隐藏生成表单同步更新。
- 未填 API Key、Base URL、Model、提示词时能阻止生成并提示。
- 参考图片上传后出现缩略图，生成时进入图生图路径。
- 批量生成过程中显示进度，并支持取消。
- 生成成功后结果区出现图片，展馆计数同步增加。
- 数据导入、清空数据不会再因 `renderApiConfigList` 报错。
- 数据导出、批量下载、自动下载不会再因 `showStatus` 缺失报错。
- 图生图结束后按钮结构仍保持，下一次生成按钮可正常显示。
