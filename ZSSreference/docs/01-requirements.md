## 1. 背景与目标

用户经常观看英文演讲 / 技术分享 / 国际会议 / 网课，语言门槛高。SimulSpeak 通过 AI 把**单向英文音频流实时翻译成中文**，以**双语字幕 + 中文配音**呈现，帮助用户跟上节奏；并具备**自动纠正之前识别/翻译错误**的能力。

目标产出：可现场跑通的 Demo —— 打开一段英文演讲，**英文原文 + 中文译文**双语字幕实时滚动、可见「自我纠错」、中文配音同步播出。

## 2. 产品定位与用户

- 定位：英文音频 → 中文字幕+配音的实时同传助手。三点差异化：**低延迟流畅**、**可见的自动纠错**、**术语一致性**。
- 用户：英语听力跟不上的中文用户（看演讲/网课/会议）。

## 3. 核心演示场景（验收主线）

1. 浏览器打开 SimulSpeak 页面，选择音频源：**标签页/系统音频**（`getDisplayMedia({audio:true})` 捕获播放中的英文演讲）或麦克风。
2. 选择纠错策略（默认「混合」），点「开始同传」→ 音频经 WebRTC 上行到 PBX 节点。
3. 字幕区出现**双语字幕**：PBX 产生英文原文（ASR）和 TMT 快翻中文，中文先以**灰色（快翻/待定）**出现。
4. 一句话说完（ASR final）后，该句**英文锁定**；主服务调用 DeepSeek Flash 基于 PBX TMT draft、上下文和术语表纠错，中文变黑锁定；若纠错与快翻不同则**高亮闪动表示「已纠正」**（核心卖点）。
5. 主服务把锁定中文作为 `tts_command` 发给 PBX，PBX 以**中文 TTS 配音**经 WebRTC 下行播放。
6. （加分）演讲结束，导出**双语 srt** + 生成**中文要点总结**。

> Demo 素材需提前精选并反复跑通：含「快翻易错、final 能纠正」的句子（专业术语、一词多义、句末否定/倒装），让「自我纠错」肉眼可见。现场用固定素材 + **录屏兜底**。

## 4. 功能需求（FR）

| 编号 | 功能 | 说明 |
|------|------|------|
| FR1 | 音频接入 | 浏览器捕获英文音频（标签页/系统/麦克风）经 WebRTC 上行 |
| FR2 | 流式英文 ASR | 英文 partial/final 识别，引擎 `16k_en` |
| FR3 | 英文原文展示 | 识别到的英文原文需展示在字幕上，而非只展示译文 |
| FR4 | 可切换纠错策略 | ① 仅快翻 ② 仅纠错 ③ 混合(默认)，运行时可选；策略控制主服务，不表示前端直接选择 PBX 内部实现 |
| FR5 | PBX 快翻草稿（final） | PBX 在 ASR final 后调用 TMT 文本翻译，出 `engine=tmt` 中文草稿 |
| FR6 | 主服务纠错（final） | ASR final→DeepSeek Flash 带 PBX TMT draft、上下文+术语表纠错，出黑色锁定中文 |
| FR7 | 自动纠错 commit/revise | final 纠错若与 PBX TMT 快翻不同则高亮回改；锁定句不再跳变 |
| FR8 | 术语一致性 | 会话级术语表，DeepSeek Flash 纠错时注入，专业术语全程统一 |
| FR9 | 双语字幕 UI | 英文原文 + 中文译文并排；灰=待定、黑=锁定、回改高亮；自动滚动 |
| FR10 | 中文 TTS 配音 | 主服务 final 中文 → PBX `tts_command` → WebRTC 下行播放 |
| FR11（加分） | 导出/总结 | 导出双语 srt/txt；演讲要点中文总结 |
| FR12 | 横向扩展 | 服务端可启动多实例，按节点分流承载更多并发会话 |

## 5. 非功能需求（NFR）

- **延迟**：TMT 中文草稿在 ASR final 后尽快返回；DeepSeek 黑色锁定（含纠错）≤ 3s；中文配音 final 后 ≤ 1s 起播。
- **稳定性**：已锁定（黑色）字幕不再变动；未锁定灰色可刷新。
- **容错降级**：PBX TMT 失败→继续显示 ASR/等待 DeepSeek Flash final；DeepSeek Flash 失败→保留 PBX TMT 快翻结果；TTS 失败→仅字幕；任一外部服务异常都不得中断字幕主链路。
- **可扩展**：翻译 provider 可插拔；服务端可水平扩容多实例。
- **Demo 可靠性**：固定素材可复现；全程录屏兜底；密钥走环境变量、不下发浏览器。

## 6. 范围边界

- **本期做**：英文→中文、单向音频、双语字幕、可切换翻译策略、自动纠错、术语一致、中文配音、多实例横向扩展（设计+最小验证）。
- **本期不做**：多目标语种（仅中文）、说话人分离、离线/本地模型、移动端、双向对话、跨节点热切换。

## 7. 优先级（MoSCoW）

- **Must**：FR1、FR2、FR3、FR5、FR9 灰→黑、基础 commit/revise。
- **Should**：FR4 策略切换、FR6 DeepSeek Flash 纠错、FR7 回改高亮、FR8 术语一致、FR10 中文配音。
- **Could**：FR11 导出/总结、FR12 多实例实测。
- **Won't**：多语种、说话人分离、离线、跨节点热切换。

## 8. 验收标准

1. 播放固定英文演讲，**英文+中文双语**字幕实时滚动，灰→黑可见。
2. 至少一处「final 纠正了快翻」的高亮回改可现场复现。
3. 三种翻译策略均可切换并产出字幕（至少混合策略完整）。
4. 中文配音与字幕大体同步播出。
5. 锁定字幕不跳变；翻译/TTS API 临时故障时字幕主链路不崩。
6. 端到端延迟符合 NFR。
7. 架构上说明并（最小）验证多实例横向扩展。
